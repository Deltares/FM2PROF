    Settings¶

    Below is a list of all parameters, categorized by which function they affect. All parameters are listed in the configuration file in the [parameters] section.

    CaseName

    Type: str | Default: Undefined | Hint: Subdirectory name under OutputDir (no space); if not specified, ‘CaseXX’ (01, 02..) are used

    MaximumPointsInProfile

    Type: int | Default: 20 | Hint: Number of points which are used to generate cross-sections

    This parameter controls how many point described cross-section geometry.

    See: Cross-section simplification algorithm

    AbsoluteVelocityThreshold

    Type: float | Default: 0.01 | Hint: Absolute velocity threshold in m/s. Used to determine storage

    These thresholds are used to distinguish conveyance (effective flow) from storage (dead zones) areas. Parts of the cross-section that do not contribute to flow are called ‘storage area’. A cell is considered part of the flow area if all of the following conditions are met:

    • the water depth is higher than 0

    • the (depth-averaged) velocity is larger than AbsoluteVelocityThreshold

    • the (depth-averaged) velocity is larger than RelativeVelocityThreshold multiplied by the average velocity in the cross-section

    These conditions are checked for each timestep in the 2D model output.

    Used in: Flow & storage seperation algorithm

    RelativeVelocityThreshold

    Type: float | Default: 0.03 | Hint: Relative velocity threshold (percent of mean flow velocity; 0-1). Used to determine storage

    See AbsoluteVelocityThreshold

    Used in: Flow & storage seperation algorithm

    MinimumDepthThreshold

    Type: float | Default: 0.02 | Hint: Minimum depth (m) for storage identification

    BedlevelCriterium

    Type: float | Default: 0.05 | Hint: Ignore the lowest percentage of bed level points

    LakeTimesteps

    Type: int | Default: 10 | Hint: Number of timesteps that are used for identifying lakes

    This parameter defines the amount of 2D model timesteps used to determine if a wet cell is hydraulically connected to the main channel. It recommended to only deviate from the default value if lakes are misidentified, and to generally keep the number low. A good test to set the value for this parameter is to visualise the ‘IsLake’ attribute (see Troubleshooting). Too high values lead to overestimation of the total volume: everything that is identified as ‘main channel’ will be used in the so-called ‘water-level independent’ part of the geometry identification.

    Used in: Lake identification algorithm

    ExtrapolateStorage

    Type: fm2prof.IniFile.ImportBoolType | Default: True | Hint: Add storage to water level independent cross-section section

    Used in Water level independent geometry algorithm

    SDCorrection

    Type: fm2prof.IniFile.ImportBoolType | Default: True | Hint: Use summerdike volume correction

    Used in Summer dike optimisation algorithm

    SDFloodplainBase

    Type: float | Default: 0.5 | Hint: minimum distance between floodplain base level and crest level in meters

    Used in Summer dike optimisation algorithm

    SDTransitionHeight

    Type: float | Default: 0.5 | Hint: Transition height at the summer dike (m)

    Used in Summer dike optimisation algorithm

    SDOptimisationMethod

    Type: int | Default: 0 | Hint: [0] Optimise on Total volume, [1] optimise on flow volume, [2] optimise on both

    Used in Summer dike optimisation algorithm

    FrictionWeighingMethod

    Type: int | Default: 0 | Hint: Options. [0] arithmetric mean, [1] Weighted average

    ExportMapFiles

    Type: fm2prof.IniFile.ImportBoolType | Default: False | Hint: Export detailed map output. Only use for small models or limited number of cross-sections b/c output can be huge.

    If this parameter is set to True, FM2PROF will output two additional geojson files. These additional files contain diagnostic information for each 2D model output (e.g. to which cross-section a 2D point is assigned). However, for large models this output can be quite large. For detailed diagnosis, combine this parameter with CssSelection

    CssSelection

    Type: fm2prof.IniFile.ImportListType | Default: Undefined | Hint: provide comma separated list of ints; e.g. 38, 39, 40. Leave empty to generate all cross-sections

    This parameter is used to run FM2PROF for a subset of cross-sections in the CrossSectionLocationFile. Its main use is for diagnostic purposes. For example, if you want to closely inspect the 54th, 76th and 89th cross-section, use:

    CssSelection = [54, 76, 89]
ExportMapFiles = True
    SkipMaps

    Type: int | Default: 0 | Hint: number of timesteps to skip at begin of 2D output. Use to skip problems related to initial conditions.

    This parameter is used to skip the first number of output timesteps (‘maps’) in the 2D model output. This parameter can be useful if the 2D model is not completely in equilibrium at the start of the computation (e.g. falling water levels in the first few timesteps). However, it tests have shown that it is far better to carefully initialize the 2D model, than to skip the first few steps with this parameter.

    ClassificationMethod

    Type: int | Default: 0 | Hint: How to classify 2D output using Section and Region polygons. Options: [0] Do not classify regions or sections. [1] Use DeltaShell

    This parameter defines the method used to classify the region and section of each output point in the 2D output. FM2PROF has built-in classification method. However these methods are not efficient for large models. The currently supported ‘DeltaShell’ method consists of a manual work-around.

    Note

    This approach is not well documented as it involves some manual work. We expect to automise this and make it more user friendly in a future update.

    MinimumTotalWidth

    Type: float | Default: 0.5 | Hint: Minimum width in meters. Zero width may lead to numerical instability in 1D solvers

    The purpose of this parameter is to prevent instabilities in 1D Solvers if they are presented with a (near) zero width. It is recommended to keep this value small (at default).
